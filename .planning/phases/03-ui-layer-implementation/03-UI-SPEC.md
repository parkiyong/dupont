# Phase 3 -- UI Design Contract

> Visual and interaction contract for the GTK4/libadwaita desktop UI layer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform note:** This is a native GTK4 desktop application using libadwaita (Adw) widgets, NOT a web application. The "design system" is Adwaita itself -- the system GTK theme that handles colors, fonts, and widget rendering for both light and dark modes automatically. All visual tokens reference GTK4 CSS classes and libadwaita widget APIs rather than hardcoded pixel values.

## Spacing Scale

GTK4/libadwaita uses property-based spacing via `margin-start`, `margin-end`, `margin-top`, `margin-bottom`, and `spacing` on containers. Values are declared in logical pixels.

Declared values (multiples of 6, following GNOME HIG baseline grid):

| Token | Value | GTK4 Property | Usage |
|-------|-------|---------------|-------|
| xs | 6px | `spacing="6"` | Icon gaps, inline padding |
| sm | 12px | `spacing="12"` | Compact element spacing, between metadata label-value pairs |
| md | 18px | `spacing="18"` | Default element spacing, between preview and metadata section |
| lg | 24px | `spacing="24"` | Section padding, between metadata and controls |
| xl | 32px | `margin-start/end="32"` | Content margins from window edges |
| 2xl | 48px | `margin-bottom="48"` | Bottom padding above controls bar |

Exceptions:
- Touch target minimum: 44px (GTK4 default button sizing satisfies this)
- Toast padding: handled by `adw::Toast` default styling (no override needed)

### Layout Structure

```
adw::ApplicationWindow
  adw::ToastOverlay
    gtk::Box (vertical, xl horizontal margins)
      adw::Clamp (max-width: 600px)
        gtk::Picture (wallpaper preview, aspect-ratio kept)
        gtk::Box (vertical, md spacing -- metadata section)
          gtk::Label (title -- heading)
          gtk::Label (description -- body)
          gtk::Label (attribution -- caption)
        gtk::Box (horizontal, lg spacing -- controls section)
          gtk::DropDown (source selector, hexpand)
          gtk::Button (refresh, with spinner)

adw::PreferencesWindow (separate window)
  adw::PreferencesPage
    adw::PreferencesGroup ("Bing")
      adw::ComboRow (market)
    adw::PreferencesGroup ("Spotlight")
      adw::EntryRow (locale)
```

## Typography

Per GNOME HIG: "Don't hard-code font styles or sizes." Use GTK4 CSS classes that map to the Adwaita type scale. The system handles sizing for light/dark mode and accessibility scaling.

| Role | GTK4 CSS Class | Weight | Line Height | Usage |
|------|---------------|--------|-------------|-------|
| Body | `.body` | Regular (400) | 1.5 (system) | Description text, window content |
| Label | (default) | Regular (400) | 1.4 (system) | DropDown labels, entry placeholders, button text |
| Heading | `.heading` | Medium (500) | 1.2 (system) | Wallpaper title, PreferencesGroup titles |
| Caption | `.caption` | Regular (400) | 1.4 (system) | Attribution text, secondary metadata |

Additional CSS classes available if needed:
- `.title-1` through `.title-4` -- larger headings (not needed in current phase)
- `.caption-heading` -- bold caption text
- `.monospace` -- for technical values (not needed in current phase)

### Typography Rules

1. All text uses GTK4 CSS classes -- never set `font-size`, `font-family`, or `font-weight` directly
2. Use `gtk::Label::add_css_class()` to apply typography roles
3. Single-line labels: set `set_ellipsize(pango::EllipsizeType::End)` and `set_lines(1)`
4. Multi-line labels: set `set_wrap(true)` and `set_wrap_mode(pango::WrapMode::WordChar)`

## Color

Adwaita system theme provides all colors automatically. The application MUST NOT set hardcoded colors. Colors adapt to light/dark mode and user theme preferences.

| Role | GTK4 Semantic | Usage | Implementation |
|------|--------------|-------|----------------|
| Dominant (60%) | `@window_bg_color` / `@view_bg_color` | Window background, content area | System default -- no override |
| Secondary (30%) | `@card_bg_color` / `@headerbar_bg_color` | Cards (if any), header bar | System default -- no override |
| Accent (10%) | `@accent_color` / `@accent_bg_color` | Interactive highlights | System default -- via widget state |
| Destructive | `@destructive_color` / `@error_color` | Error states | System default -- for toast styling |

### Accent Reserved For

Accent color is reserved for these specific elements ONLY:
1. **Refresh button** -- `gtk::Button` with `.suggested-action` CSS class (filled accent button)
2. **Active Dropdown item** -- `gtk::DropDown` selected item highlight (system default)

All other widgets use default (non-accent) styling:
- Source dropdown: default `gtk::DropDown` (no accent class)
- Settings window elements: default `adw::EntryRow` / `adw::ComboRow`
- Labels and metadata: neutral text colors (system default)

### Error Styling

- Error toasts use `adw::Toast` default styling (no custom color needed)
- Optionally add `.error` CSS class to toast for destructive/error emphasis if libadwaita supports it (verify during implementation)

## Interaction Patterns

### State Machine

The UI component model tracks these states:

| State | Visual | Trigger |
|-------|--------|---------|
| **Idle -- no wallpaper** | Empty state (placeholder icon + text) | Application launch before first fetch |
| **Loading** | Spinner visible in refresh button, dropdown disabled | User clicks Refresh or switches source |
| **Loaded** | Wallpaper preview + metadata displayed | Successful fetch from source |
| **Error** | Toast notification, previous content preserved | Any failure (network, parse, cache, DE) |

### Async Flow

```
User clicks "Refresh Wallpaper"
  -> Component sends Command { async { source.fetch().await } }
  -> Component model sets state = Loading
  -> UI: spinner starts, dropdown disabled
  -> Command completes with Ok(Wallpaper) or Err(SourceError)
  -> Ok: cache.get_or_download().await -> display preview
  -> Err: add Toast to ToastOverlay with error message
```

### Loading State Decision

**Spinner in refresh button** (Claude's discretion, D-02). The refresh button label changes to a `gtk::Spinner` during loading. This is non-blocking -- the user can still interact with the window. The source dropdown is disabled (not the whole window) to prevent source-switch race conditions.

### Toast Specification

| Property | Value | Source |
|----------|-------|--------|
| Position | Bottom center (adw::ToastOverlay default) | D-04, GNOME HIG |
| Duration | 5 seconds auto-dismiss | Claude's discretion |
| Dismissal | Click to dismiss immediately | Standard adw::Toast behavior |
| Detail button | None for v1 (no undo action) | MVP scope |
| Max visible | 1 toast at a time (new toast replaces previous) | Simplicity for v1 |

### Toast Error Messages

Mapped from domain error types:

| Error Type | Toast Title | Toast Body |
|------------|-------------|------------|
| `SourceError::HttpError` | "Connection failed" | Display `SourceError::Display` message |
| `SourceError::ParseError` | "Invalid response" | "The wallpaper source returned an unexpected format" |
| `SourceError::NoWallpaperFound` | "No wallpaper available" | "The source did not return a wallpaper" |
| `SourceError::Unavailable` | "Source unavailable" | Display `source_name` from error |
| `SourceError::RateLimited` | "Rate limited" | Display `source_name` from error |
| `CacheError::IoError` | "Cache error" | Display `CacheError::Display` message |
| `CacheError::ImageError` | "Invalid image" | "The downloaded image could not be decoded" |
| `CacheError::NotAccessible` | "Cache error" | "Cache directory is not accessible" |
| `DEError::SetError` | "Failed to set wallpaper" | Display `DEError::Display` message |
| `DEError::UnsupportedDE` | "Unsupported desktop" | Display DE name from error |
| `DEError::DetectionFailed` | "Desktop detection failed" | "Could not detect the current desktop environment" |

General fallback for unhandled errors: "Something went wrong" with the error's Display message as body.

## Copywriting Contract

Per GNOME HIG writing style: sentence capitalization for labels (capitalize first word only), no trailing periods on labels/titles, concise and action-oriented.

| Element | Copy | Source |
|---------|------|--------|
| Window title | "Damask" | Application name |
| Primary CTA (idle state) | "Refresh Wallpaper" | UI-03, GNOME style (verb + noun) |
| Primary CTA (loading state) | (spinner replaces text) | Interaction pattern |
| Primary CTA (error recovery) | "Try Again" | Displayed after error toast, same button |
| Source dropdown label | "Source" | UI-04 |
| Source options | "Bing", "Spotlight" | Source::name() from domain |
| Empty state heading | "No wallpaper loaded" | UI-01, Claude's discretion |
| Empty state body | "Select a source and refresh to load your first wallpaper" | UI-01, Claude's discretion |
| Metadata: title | `wallpaper.title` | UI-02, direct from domain |
| Metadata: description | `wallpaper.description` | UI-02, direct from domain |
| Metadata: attribution | `wallpaper.attribution` | UI-02, direct from domain |
| Settings window title | "Damask Preferences" | UI-06 |
| Settings group: Bing | "Bing" | UI-06 |
| Settings row: Bing market | "Market" (AdwComboRow title) | UI-06 |
| Settings group: Spotlight | "Spotlight" | UI-06 |
| Settings row: Spotlight locale | "Locale" (AdwEntryRow title) | UI-06 |
| Error state (toast title) | See Error Messages table above | UI-05, mapped from domain errors |
| Error state (toast body) | See Error Messages table above | UI-05 |
| Destructive confirmation | None in v1 -- no destructive actions in this phase | Out of scope |

### Copywriting Rules

1. **Sentence capitalization** for all labels: "Refresh wallpaper", not "Refresh Wallpaper" -- EXCEPT proper nouns "Bing" and "Spotlight" remain capitalized
2. **No trailing periods** on labels, titles, button text, or dropdown items
3. **Description/body text** may use periods for complete sentences
4. **Error messages** use the `Display` impl from domain error types directly -- do not rephrase
5. **Metadata fields** display empty string (not "N/A" or "Unknown") when domain returns empty
