## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation |
|-----------|----------|-----------|----------|-------------|------------|
| T-03-01 | S (Spoofing) | Wallpaper URL from API | info | accept | URL comes from hardcoded API endpoints (bing.com, arc.msn.com). No user-supplied URLs. gtk::Picture renders via GDK-Pixbuf which handles malformed images safely. |
| T-03-02 | T (Tampering) | Cache file path | info | accept | Cache filenames derived from wallpaper.id (API-controlled, not user-controlled). Cache dir is ~/.cache/damask/ (standard XDG). No path traversal vector. |
| T-03-03 | R (Repudiation) | N/A | info | accept | Desktop app with no audit requirements. |
| T-03-04 | I (Info Disclosure) | Error toast messages | low | accept | Toast messages use Display impl of domain errors which may include API URLs. Acceptable -- user can see the same URLs in browser. No PII in any error messages. |
| T-03-05 | D (Denial of Service) | Malformed image in cache | info | accept | Domain's validate_and_detect() uses image::Limits (8192x8192, 50MB max alloc). Already mitigated in Phase 1 (Cache implementation). |
| T-03-06 | E (Elevation of Privilege) | N/A | info | accept | No privilege boundaries in a single-user desktop app. |
