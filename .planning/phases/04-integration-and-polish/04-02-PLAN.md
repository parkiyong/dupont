<objective>
## Goal
Create a desktop entry file and app icon so Damask appears in GNOME/COSMIC application launchers.

## Purpose
Without a `.desktop` file and icon, the app can only be launched from the terminal. Desktop integration makes it discoverable and feel like a native Linux application.

## Output
- `.desktop` file at `data/com.damask.Wallpaper.desktop`
- Scalable icon (SVG) at `data/icons/scalable/com.damask.Wallpaper.svg`
- Symbolic icon (SVG) at `data/icons/symbolic/com.damask.Wallpaper-symbolic.svg`

<task type="auto">
  <name>Create desktop entry file</name>
  <files>data/com.damask.Wallpaper.desktop</files>
  <action>
    Create `data/com.damask.Wallpaper.desktop` following the Desktop Entry Specification:

    ```ini
    [Desktop Entry]
    Type=Application
    Name=Damask
    Comment=Set your desktop wallpaper from Bing and Spotlight
    Exec=damask-app
    Icon=com.damask.Wallpaper
    Categories=GNOME;GTK;Utility;DesktopSettings;
    Keywords=wallpaper;background;bing;spotlight;
    StartupNotify=true
    StartupWMClass=com.damask.Wallpaper
    ```

    Notes:
    - Categories: GNOME and GTK for DE filtering, Utility for general, DesktopSettings for settings-like behavior
    - StartupWMClass matches the Relm4 app ID "com.damask.Wallpaper"
    - Exec will need to be the full path or in $PATH when installed — document in SUMMARY
  </action>
  <verify>File is valid Desktop Entry format (check with `desktop-file-validate` if installed, or manual review of fields)</verify>
  <done>AC-1 satisfied: Desktop file follows spec with correct fields</done>
</task>
